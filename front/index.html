Emulator Results

These are the results of a RAFT Leader Election process simulation which displays different properties.

  • raft_messages_count: refers to the number of messages generated by the whole network during the process.
  • raft_pings_count: refers to the number of pings generated by the leader after elected to keep informed to the followers.
  • raft_messages_size: refers to the size the messages generated by every node.
  • raft_average_medium_time: refers to the average time (nanoseconds) a message spends in the medium or in transmission to reach all nodes.
  • raft_winner_count: the amount of winners per each round. This value should always be 1, if is more than 1 it means that there is a case where the algorithm fails.
  • raft_election_time_ms: refers to the time (milliseconds) the process takes to converge, from starting the process to converging in a leader.
  • raft_buffer_channel_time: refers to the average time (milliseconds?) the application takes to analyze each packet.
